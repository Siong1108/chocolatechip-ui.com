Paging Control

ChocolateChip-UI offers the pager control as a way to provide horizontal swipe navigation through a limited set of panels. This is analogous to the iOS paging control with dots beneath it to indicate each page and were the user is in paging progress. Be aware that because of the limited screen width of mobile devices in portrait mode, you should try to fit more than 17 panels in a paging control. If you need to provide paging for more than that then you should use a segmented paging control.

The paging control uses a stack with the attribute ui-implements="paging". The paging stack contains two nest children: a panel and another stack to hold the paging content. This nesting is required to setup the environment to implement this type of control properly. Each individual section of the paging control is a panel. Below is a typical paging control stack:

Example:

<stack ui-implements="paging">
    <panel>
        <stack>
            <panel>
                <h4>Panel 1</h4>
                <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod.</p>
            </panel>
            <panel>
                <h4>Panel 2</h4>
                <p>Sed ut perspiciatis unde omnis iste natus error sit voluptatem accusantium doloremque laudantium.</p>
            </panel>
            <panel>
                <h4>Panel 3</h4>
                <p>Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt.</p>
            </panel>
            <panel>
                <h4>Panel 4</h4>
                <p> Nemo enim ipsam voluptatem quia voluptas sit aspernatur aut odit aut fugit, sed quia consequuntur magni dolores eos qui ratione voluptatem.</p>
            </panel>
        </stack>
    </panel>
</stack>

In the above example we also use panels to indicate the scrollable sections of the paging control. These have their styles set by default, but you can override them in your app's local stylesheet. When you app loads, ChocolateChip-UI sees the stack structure above and initializes it to work as a horizontal paging control. It check to see how many pagable panels are in the control and creates a paging indicator beneath it with the appropriate number of dot indicators. As the user swipes left and right, the appropriate page indicator will get highlighted.

If you want to create a different sized paging panel, you can adjust the dimensions here (Note: the if you change the width of the paging stack, you'll also need to make the paging indicators stack the same width):

stack[ui-implements="paging"] > panel > stack > panel {
    width: 320px; 
    height: 280px;
    text-align: center;
    font-size: 18px;
}
stack[ui-implements="indicators"] {
    width: 320px;
}

To change the default background color of the paging control stack and the paging indicator stack, redefine these values:

stack[ui-implements=paging], stack[ui-implements=indicators] {
    background-color: #fff !important;
}

To customize the default colors of the paging indicators, change the colors of these in your app's local stylesheet:

stack[ui-implements="indicators"] > indicator {
    width: 8px; 
    height: 8px;
    -webkit-border-radius:4px;
    background: #ddd;
}
stack[ui-implements="indicators"] > indicator.active {
    background: #888;
}

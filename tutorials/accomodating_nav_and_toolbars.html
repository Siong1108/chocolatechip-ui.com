Accomodating Navbars, etc.

ChocolateChip-UI provides a way to help you manage your layouts so that things are always positioned as you want, even when the user changes the orientation of the mobile device. It does this by using various sets of tags to indicate relationships. The holy grail of layouts is one with a navbar along the top of the app, a subview with scrollable content and a toolbar along the bottom. If you looked at the previous tutorial "Basic App Shell" you already so an example of how this is done. But you might want some variation on this. There are a number of possible ways in which you might want to use a subview in your app. They're listed below.

  • just a subview
  • a subview with a navbar along the top
  • a subview with a navbar along the top and a toolbar along the bottom
  • a subview with a toolbar along the top
  • a subview with a toolbar along the top and a toolbar along the bottom
  • a subview with a toolbar along the bottom

All of these variations are easy to achieve by use of the ui-associations attribute on the subview.

Just a subview

In this case, just put the subview into the view and you're done.

A subview with a navbar along the top and a toolbar along the bottom

Give the subview this attribute: ui-associations="withNavBar withBottomToolBar".

A subview with a toolbar along the top

Give the subview this attribute: ui-associations="withTopToolBar".

A subview with a toolbar along the top and a toolbar along the bottom

Give the subview this attribute: ui-associations="withTopToolBar withBottomToolBar".

A subview with a toolbar along the bottom

Give the subview this attribute: ui-associations="withBottomToolBar".

Note: Be sure to put a scrollpanel inside the subveiw to hold your data. That way the user will be able to access data that overflows the available screen dimensions.

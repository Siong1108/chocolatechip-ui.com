UIButtons

ChocolateChip-UI provides a number of different types of uibuttons. The most common uibutton is found in the navbars and toolbars. It typically has a bluish gradient and a rounded border.

The uibuttons types are:

  • Default
  • Navigation UIButtons: Back and Next
  • UIButton with Icon and Label
  • UIButton with Icon but no Label
  • UIButton with Icon and no Label or Border
  • Rounded Rectangle UIButton
  • Action UIButton

Here is how these different buttons look. Some of the icons have custom background colors or gradients applied using the techniques described below.

Default:

ChocolateChip-UI's UIButtons is very simple: a uibutton with a label. Even if you create one dynamically, its interaction states for hover and touch are automatically provided by ChocolateChip.

Example:

<uibutton>
    <lable>Click Me!</label>
</uibutton>

 

Navigation UIButtons: Back and Next:

If you need a "back" or "next" uibutton, give the button the attribute ui-implements="back" or ui-implements="next". ChocolateChip-UI will automatically create the appropriate pointer for the uibutton.

Example:

<uibutton ui-implements="back">
    <label>Back</label>
</uibutton>

<uibutton ui-implements="next">
    <label>Next</label>
</uibutton>

 

UIButton with Icon and Label:

ChococlateChip-UI lets you include an icon in your buttons. You do this by putting an icon tag in the button. There are two ways to use and image for an icon: you can use the image as a background image or as an image mask. In the case of a high fidelity pixel image, you can use it as a background image on the icon for best results. To do so you put an inline style directly on the icon. Or, if you prefer, you could put a unique id on the button and create a CSS definition in your document to accomplish the same thing.

Example:

<uibutton ui-icon-align="right" ui-bar-align="right">
    <icon  style="background-image: url(icons/iTunes_icon.png);"></icon>
    <label>iTunes</label>
</uibutton>

ChocolateChip-UI provides over forty common icons in SVG format for use as image masks. The advantage of these is that they are resolution independent. You can find them in the icons folder. Because they were designed for use as masks, they are black with transparency. By default ChocolateChip-UI sets a white background color on icons with image masks. You you change this in an inline style to whatever color you want, allowing for some interesting effects. As a matter of fact, you could even use a CSS gradient as the background for the image mask. Below are examples of the default, with a background color and with a gradient. To implement the image mask, use the -webkit-mask-image property with a url pointing to the image to use as the mask.

Example:

<!-- Default white background for image mask: -->
<uibutton>
    <icon style="-webkit-mask-image: url(icons/attachement.svg);"></icon>
    <label>Save</label>
</uibutton>
<!-- An icon with a custom background color and image mask: -->
<uibutton>
    <icon style="-webkit-mask-image: url(icons/add.svg); background-color: #fff5b6;"></icon>
    <label>Add</label>
</uibutton>
<!-- An icon with a CSS gradient and image mask: -->
<uibutton>
    <icon style="-webkit-mask-image: url(icons/cog.svg) background-image: -webkit-gradient(linear, left top, right bottom, from(yellow), to (orange));"></icon>
    <label>Settings</label>
</uibutton>

UIButton with Icon but no Label:

Sometimes you want your pretty little button to show an icon but no label. ChocolateChip-UI makes this possible by putting the ui-implements="icon" attribute on your button and providing an image mask on the icon. You can use the same techniques for adding background color describe above.

Example:

<!-- Default white background for image mask: -->
<uibutton ui-implements="icon">
    <icon style="background-image: url(icons/files.svg);"></icon>
</uibutton>
<uibutton  ui-implements="icon">
    <icon style="-webkit-mask-image: url(icons/add.svg); background-color: #fff5b6;"></icon>
</uibutton>
<uibutton ui-implements="icon">
    <icon style="-webkit-mask-image: url(icons/cog.svg)" background-color: yellow;"></icon>
</uibutton>

 

UIButton with Icon and no Label or Border:

A common type of uibutton for toolbars in particular has an icon, but the rest of the uibutton is transparent. This type of button has no label either, it is purely an icon. ChocolateChip-UI makes this type of uibutton possible with the ui-kind="icon" attribute. Remember to leave the labels out. You can use any of the background coloring techniques outlined above.

Example:

<!-- Default white background for image mask: -->
<uibutton ui-kind="icon" class="disabled">
    <icon style="-webkit-mask-image: url(icons/film.svg);">
</uibutton>
<!-- An icon with a custom background color and image mask: -->
<uibutton ui-kind="icon">
    <icon style="-webkit-mask-image: url(icons/files.svg); background-color: #97611f;"></icon>
</uibutton>
<!-- An icon with a CSS gradient and image mask: -->
<uibutton ui-kind="icon">
    <icon style="-webkit-mask-image: url(icons/download.svg); background-image: -webkit-gradient(linear, left top, right bottom, from(yellow), to (orange));">
</uibutton>

 

Rounded Rectangle UIButton:

Rounded rectangle uibuttons are the simplest in style: just a plain white background a a simple rounded border. This style of uibutton is common in subview outside of tableviews. By default thiese buttons stretch to the full available width of their container so that they stack up on top of each other.

Example:

<uibutton ui-kind="rounded-rectangle">
    <label>Get Update</label>
</uibutton>
<uibutton ui-kind="rounded-rectangle">
    <label>Text Message</label>
</uibutton>

If you want to change the stretching and stacking behavior of the rounded-rectangle uibuttons so that they sit side by side, you can simply put them in a stack container. Since stack has a default box orientation of vertical, you'll need to override that by giving it ui-box="horizontal".

Example:

<stack ui-box="horizontal">
    <uibutton ui-kind="rounded-rectangle">
        <label>Get Update</label>
    </uibutton>
    <uibutton ui-kind="rounded-rectangle">
        <label>Text Message</label>
    </uibutton>
</stack>

By default the rounded rectangle uibuttons have bluish text. You could redefine this in you document's stylesheet to whatever you what. ChocolateChip-UI does offer the standard iOS secondary rounded rectangle uibutton alternative using ui-implements="important" to give the rounded rectangle uibutton black text.

 

Action UIButton

Action uibuttons are used in action sheets, popups and popovers. Action uibuttons are larger and more colorful than other buttons. They're meant to be larger targets and call out the attention of the user to choose one of several tasks to perform. ChocolateChip offers a number of default styles for action uibuttons:

  • class="default" - the default gray
  • class="cancel" - a charcoal/black
  • class="done" - blue
  • class="delete" - red
  • class="green" - green
  • class="gold" - gold

The appearance of an action uibutton is triggered by the ui-kind="action" attribute. You can disable an action uibutton by giving it a class of "disabled".

<uibutton ui-kind="action" ui-implements="default">
    <label>Button</label>
</uibutton>

<uibutton ui-kind="action" ui-implements="cancel">
    <label>Button</label>
</uibutton>

<uibutton ui-kind="action" ui-implements="done">
    <label>Button</label>
</uibutton>

<uibutton ui-kind="action" ui-implements="delete">
    <label>Button</label>
</uibutton>

<uibutton ui-kind="action" class="green">
    <label>Button</label>
</uibutton>

<uibutton ui-kind="action" class="gold">
    <label>Button</label>
</uibutton>

<uibutton ui-kind="action" class="disabled">
    <label>Button</label>
</uibutton>

 

Resolution Independence

The interfaces you create with ChocolateChip-UI's WAML and CSS are resolution independent. You never have to worry about the pixel density on the target device. Icon Alignment:

You can tell ChococlateChip-UI how to align icons in your uibuttons in relation to their labels using the ui-icon-align attribute. By default all icons get aligned to the left of a uibutton's label. You do not have to do anything. To set the alignment of a uibutton's icon you can choose any of these values:

  • left
  • top
  • right
  • bottom

Example:

<uibutton ui-kind="icon" ui-icon-align="left">
    <icon style="-webkit-mask-image: url(icons/film.svg);">
</uibutton>
<uibutton ui-kind="icon" ui-icon-align="right">
    <icon style="-webkit-mask-image: url(icons/files.svg); background-color: #97611f;"></icon>
</uibutton>
<uibutton ui-kind="icon" ui-icon-align="top">
    <icon style="-webkit-mask-image: url(icons/cog.svg);"></icon>
</uibutton>
<uibutton ui-kind="icon" ui-icon-align="bottom">
    <icon style="-webkit-mask-image: url(icons/download.svg); background-color: yellow">
</uibutton>

Note:

Generally, when aligning icons on buttons in navbars and toolbars you should stick to left and right alignment. Buttons in the bars do not have enough vertical space to handle vertical alignment of icons. The same goes for segmented controls in navbars and toolbars. Segmented controls placed outside a toolbar directly in a subview and action buttons have enough vertical space to handle top/bottom alignment of icons.

 

UIButton Placement in Navbars and Toolbars

You can control the placement of buttons in navbars and toolbars as well.

Navbar Placement

ChocolateChip allows you to control the placement of buttons and segmented controls in a navbar. By default a navbar centers its content. This means that if all you have is an H1 for the title, it gets centered. This also centers any buttons or segmented controls you put in a navbar. This doesn't work if you want a title plus a button on the left or right or both. To place buttons or other items on the sides you can give them the ui-bar-align attribute. See the example below:

Example:

<navbar>
    <uibutton ui-implements="back" ui-bar-align="left">
        <pointer></pointer>
        <label>Back</label>
    </uibutton>
    <h1>Home</h1>
    <uibutton ui-implements="next" ui-bar-align="right">
        <pointer></pointer>
        <label>Next</label>
    </uibutton>
</navbar>

Note: For more information about segmented controls, see the tutorial and consult the documentation.

Toolbar Placement

By default all toolbars arrange their contents evenly spaced. You can override this by using the same ui-bar-align attribute on items in a toolbar. You can also use any of the various box attributes which ChocolateChip-UI provides, such as:

Example:

Toolbar with buttons align to the left and right:

Toolbar with all buttons packed to the right:

<toolbar ui-box-pack="end">
    <uibutton>
        <label>Click 1</label>
    </uibutton>
    <h1>Home</h1>
    <uibutton>
        <label>Click 2</label>
    </uibutton>
    <uibutton>
        <label>Click 3</label>
    </uibutton>
</toolbar>

 

Disabling Buttons:

It's easy to disable any ChocolateChip-UI button. Just put a class "disabled" on it. This will change the appearance of the button and signal to ChocolateChip-UI that it should not listen for events. Depending on how you've set up your own events, you may need to add in some conditional checks to see if a uibutton has a class of "disabled" to avoid firing and event. This is easy to do.

Example:

$("uibutton.specialKindaGuy").bind("touchstart", function() {

// Check to see if the uibutton is disabled or not.
if (!this.hasClass("disabled")) {
    // Do stuff here.
}

})

 

Creating UIButtons Dynamically:

You can create uibuttons dynamically using several methods. The easiest way is to use the $.insert method. You pass it a string of the button you want to create and insert it in the designated container. You could also use the ECMAScript5 insertAdjacentHTML method.

Example:

var myButtons = '<uibutton ui-bar-align="right">\
    <icon style="-webkit-mask-image: url(icons/files.svg); \
    background-color: #97611f;"></icon>\
    <label>Keep</label>\
    </uibutton>';

// Convert string into HTML and insert it in the
// last position of the third navbar:
$("#navbar:nth-of-type(3)").insert(myButton, "last");

Note: When composing strings as above, you can use a backslash to add a newline and continue the string without braking it. This makes you string more readable and maintainable.

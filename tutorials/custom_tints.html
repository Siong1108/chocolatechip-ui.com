Custom Tints

Although ChocolateChip-UI provides default styles and some variations for its controls, we also understand the need to allow custom styles for branding and personalization of an app. To accomplish this you can take advantage of custom tints. Using the ui-custom-tint class to override the default style. Custom tints automatically provide hover states for uibuttons. Custom tints work with the following elements:

  • navbar
  • toolbar
  • uibutton default
  • uibutton segmented control
  • uibutton action
  • uibutton rounded rectangle
  • popup
  • action sheet
  • switch control
  • background gradients

The ui-custom-tint class overrides the default gradient on UI elements and replaces it with a gray scale gradient with degrees of opacity. This allows a background color to show through. So, once the ui-custom-tint class is applied to an element, all you have to do is apply a background color.

 

Navbar/Toolbar:

You can customize the background color of navbars and toolbars using ui-custom-tint and an inline background color or additional class or CSS3 selector. To change the background color on all of the navbars and toolbars in your app, you could do the following:

Example:

navbar.ui-custom-tint, toolbar.ui-custom-tint { background-color: lightblue; }

Or you might want to use a selector to target a specific navbar/toolbar:

#view_03 > navbar.ui-custom-tint {
    background-color: #777;
}
view:first-of-type > toolbar.ui-custom-tint {
    background-color: orange;
}
view.ui-custom-tint.green {
    background-color: green;
}

Or you could apply the background-color inline on the bar:

<navbar class="ui-custom-tint" style="background-color: #d3beab;">
    <button ui-implements="back">
        <label>Back</label>
    </button>
    <h1>Latest Events</h1>
</navbar>

 

UIButton (default):

Like navbars and toolbars, it's very easy to change the look of the default gradient and background color of uibuttons in navbars and toolbars. Just put the ui-custom-tint class on the uibuttons and give them a background color. You can use any of the methods illustrated above for the navbars/toolbars:

/* An RGBA background color with transparency allowing 
the background to show through the buttons.*/
navbar > uibutton.ui-custom-tint {
    background-color: rgba(211, 17, 19, 0.65); 
}

#main > toolbar > uibutton[ui-implements=icon].ui-custom-tint {
    background-color: #6f8000;
}

Note: If you plan on using a custom tint on a Back or Next uibutton, you'll also need to put a background color on those uibuttons' pointers. The CSS definition below expects that the class ui-custom-tint will come after the ui-implements attribute. If you're not sure about which order your uibuttons will have, provide both orders of values (attribute-class/class-attribute):

uibutton[ui-implements=back].ui-custom-tint,
uibutton[ui-implements=back].ui-custom-tint > pointer {
    background-color: #51b3d3;
}

<button ui-implements="back" class="ui-custom-tint">
    <label>Back</label>
</button>

 

UIButon (segmented control):

The uibuttons in segmented controls can also have their background colors changed using the ui-custom-tint class. Since segmented controls have two default styles: in navbars/toolbars and in a view/subview, you can style your custom colors for both:

navbar > segmentedcontrol > uibutton.ui-custom-tint {
    background-color: #ffdf7f;
}
subview > segmentedcontrol > uibutton.ui-custom-tint {
    background-color: #c0a290;
}

 

UIButton (action):

Action uibuttons are mostly for use in action sheets and popups. ChocolateChip-UI already provides several possible colors for action uibuttons:

  • implements="default": the default gray
  • implements="delete": red
  • implements="done": blue
  • implements="cancel": black
  • class="green": green
  • class="gold": gold

Like regular uibuttons, it's easy to create a custom color for action uibuttons using the same techniques as for regular uibuttons. Put a class of ui-custom-tint on the action uibutton, then give it a background color. You can use any of the techniques described for default uibuttons to accomplish this: inline style declaration, additional class definition or a CSS3 selector definition. Note: An inline style will always override a class definition. See example below:

uibutton[ui-kind="action"][ui-implements="default"].ui-custom-tint {
    background-color: #8fc060;
}

<uibutton ui-kind="action" ui-implements="default" class="ui-custom-tint">
    <label>Set Value</label>
</uibutton>
<uibutton ui-kind="action" ui-implements="default" class="ui-custom-tint" style="background-color: #ff6666">
    <icon style="-webkit-mask-image: url(icons/location_pin.svg);"></icon>
    <label>Set Value</label>
</uibutton>

 

UIButton (Rounded Rectangle)

Rounded rectangle uibuttons can also be customized, even though they don't have background gradients. Maybe you do want them to have a background gradient, or you want to change the font color, border color or background color. You can easily use some class definitions to do this in your document's local stylesheet:

uibutton[ui-kind="rounded-rectangle"].ui-custom-tint {
    border: red;
    border-width: 2px;
    color: gold;
    background-color: orange
}

 

Popup:

You can customize the look of popups as well, see the section above about action uibuttons for how to customize them. Besides customize the theme of the popup's action uibuttons, you might also want to customs the look of the popup itself. Even the though popup is created dynamically, it's style are in ChUI.css so you can override them. You might want to change the background color to black or something else, or remove the glow behind the popup title. The popup's background is defined for the popup element, the radial gradient is defined for the panel element inside the popup. So to handle both you could do something like this:

popup { 
background-image: 
    -webkit-gradient(linear, left top, left bottom,
        from(rgba(128,0,45,0.5)),
        to(rgba(98,0,35,0.5)));
}

popup > panel {
    background-image: none
}

 

Action Sheet:

Like popups, you can customize the looks of an action sheet's action uibuttons same as for any other action uibuttons. You can also customize the sheet itself. By default it has a bluish tint, but you can at initialization time choose to have it displayed with a black gradients. Refer to the documents in ChUI.js for actionsheet. Maybe you want a totally different color. You can do that by overriding the action in your document's local stylesheet:

/* Define a brown background for the action sheet: */
actionsheet    {
    background-image:
        -webkit-gradient(linear, left top, left bottom,
            from(#000),
            color-stop(0.5, #000),
            color-stop(0.5, rgba(225,225,225,.6)),
            to(rgba(225,225,225,.6))),
        -webkit-gradient(linear, left top, left bottom,
            from(rgba(225,225,225,.5)),
            color-stop(0.9, rgba(225,225,255,.15)),
            to(rgba(225,225,225,.015))),
        -webkit-gradient(linear, left top, left bottom,
            from(rgba(128,48,0,0.45)),
            to(rgba(128,48,0,0.65))) !important;
}

 

Switch Control:

To customize the "ON" label of a switch control, just add a class to the switch and define a color for it in your document's local stylesheet:

switchcontrol.customSwitchColor > label[ui-implements=on] {
    border: solid 1px #005800;
    background-image: 
        -webkit-gradient(linear, left top, left bottom, 
            from(#923700), 
            color-stop(.5, #c04800 ));
}

 

Background Gradients:

ChocolateChip-UI provides the standard iOS vertically striped background gradient, as well as a number of transparent gradient patters to use with any background color you choose. However, you could also take one of the gradient patterns and customize in in your app's local stylesheet to give you app a truly unique appearance. You might want equal width stripes. You might want the pattern to have a translucent color to overlay the background color. You decide.

Here's the slant-right pattern with equal width stripes:

body[ui-background-style=slanted-right] {
    background-color: #d4e0f3;
    background-image: 
    -webkit-gradient(linear, left top, right bottom,
        color-stop(0.25, rgba(0,0,0,0.1)), 
        color-stop(.3, transparent),
        color-stop(0.5, transparent), 
        color-stop(.5, rgba(0,0,0,0.1)),
        color-stop(0.75, rgba(0,0,0,0.1)), 
        color-stop(.8, transparent),
        to(transparent));
/* Adjust the background size to change the thickness of the stripes: */
-webkit-background-size: 15px 15px;
background-size: 15px 15px;

Here's the slant-left pattern with equal width stripes and a translucent stripe color over the background color:

body[ui-background-style=slanted-right] { background-color: #d4e0f3; background-image: -webkit-gradient(linear, left top, right bottom, color-stop(0.25, rgba(145,127,255,0.3)), color-stop(.3, transparent), color-stop(0.5, transparent), color-stop(.5, rgba(145,127,255,0.3)), color-stop(0.75, rgba(145,127,255,0.3)), color-stop(.8, transparent), to(transparent)); -webkit-background-size: 40px 40px; background-size: 15px 15px;
}

 

Customizing Custom Tint

That's right, you can even customize custom tint. Don't like the way the gradient is, change the stop positions, change the gradient color from black to some other color that will be composited over the background color. If you have design sensibilities or you have a designer at your side, experiment with this and see what you come up with. Be aware that custom tint has a hover state consisting of the same gradient reversed, so if you change the default gradient, you'll probably want to change that one as well. By the way, like all customized styles, copy the original and work on it in your document's local stylesheet, not in ChUI.css.

Example:

navbar > uibutton,
toolbar > uibutton {
background-image: 
    -webkit-gradient(linear, left top, left bottom, 
        from(rgba(255,255,255,.5)), 
        color-stop(0.6,rgba(30,30,30,.5)),
        color-stop(0.6, rgba(0,0,0,.5)), 
        to(rgba(0,0,0,.75)));
background-color: orange;
}
navbar > uibutton.ui-custom-tint:hover,
toolbar > uibutton.ui-custom-tint:hover {
    background-image: 
        -webkit-gradient(linear, left top, left bottom, 
            from(rgba(0,0,0,.1)), 
            color-stop(0.45,rgba(0,0,0,.5)), 
            color-stop(0.45, rgba(0,0,0,.6)), 
            to(rgba(255,255,255,.2)));
}

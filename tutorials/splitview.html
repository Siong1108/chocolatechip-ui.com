Split View

ChocolateChip-UI provides the split view layout for tablets that provides the basic functionality of a left navigation section and a large right detail view. There is actually nothing to do to initialize the split view layout: just provide the correct layout and ChocolateChip-UI will take care of the rest. A split view layout looks like this:

<app>
    <splitview>
        <rootview>
            <panel>
                <view id="main" ui-navigation-status="current">
                    <navbar>
                        <h1>Navigation</h1>
                    </navbar>
                    <subview id="scroller1">
                        <scrollpanel>
                        </scrollpanel>
                    </subview>
                </view>
                <view id="second" ui-navigation-status="upcoming">
                    <navbar>
                        <uibutton ui-implements="back" ui-bar-align="left">
                            <label>Back</label>
                        </uibutton>
                        <h1>Pick One</h1>
                    </navbar>
                    <subview id="scroller2">
                        <scrollpanel>
                        </scrollpanel>
                    </subview>
                </view>
            </panel>
        </rootview>
        <detailview>
            <navbar>
                <h1>Detail View</h1>
            </navbar>
            <subview id="scroller2">
                <scrollpanel>
                </scrollpanel>
            </subview>
        </detailview>
    </splitview>
</app>

In the above split view notice that the root view has two views as children. The first would hold a typical drill down list of a tableview with detail disclosures that reveal another view. You can populate the second view with data based on the user's choice on the first view. The root view's second view would then be an action list for performing actions on the detail view. When implementing a root view with navigation of views, remember to give the ui-navigation-status values just like normal navigation lists in ChocolateChip-UI: "current" or "upcoming".

Of course, you may not need a drill down list in the root view. In that case the root view can have just one view with actionable list or uibuttons to affect the data in the detail view.

When there is an orientation change to portrait mode, the split view hides the root view and shows a button in the detail view's header with the name of the hidden root view. This button will toggle the visibility of the root view, which in portrait mode appears as a popover. When the user turns the tablet back to landscape mode, the popover get converted back to its normal root view layout.

If you’d prefer to not have the rootview hidden in portrait mode, you can add a class of “SplitViewFixed” to your app’s body tag. Then the rootview and detailview will remain visible in all orientations, with the detail view resizing depending on the amount of width available to it.

Example:

<body class="SplitViewFixed">
    <app>
        <splitview>
            <rootview>
                <panel>
                    <view id="main" ui-navigation-status="current">
                        <navbar>
                            <h1>Navigation</h1>
                        </navbar>
                        <subview id="scroller1">
                            <scrollpanel>
                            </scrollpanel>
                        </subview>
                    </view>
                    <view id="second" ui-navigation-status="upcoming">
                        <navbar>
                            <uibutton ui-implements="back" ui-bar-align="left">
                                <label>Back</label>
                            </uibutton>
                            <h1>Pick One</h1>
                        </navbar>
                        <subview id="scroller2">
                            <scrollpanel>
                            </scrollpanel>
                        </subview>
                    </view>
                </panel>
            </rootview>
            <detailview>
                <navbar>
                    <h1>Detail View</h1>
                </navbar>
                <subview id="scroller2">
                    <scrollpanel>
                    </scrollpanel>
                </subview>
            </detailview>
        </splitview>
    </app>
</body>
